fixUXV1.1
QA caught it: after scoring the ICP, booking a meeting, etc., the Gate Progress + "Should we walk?" cards stayed stale until a hard refresh. Root cause: query keys are single-string URLs (e.g. "/api/deals/:id/transition-check?toStageCode=engaged"), so invalidateQueries(["/api/deals/:id"]) only prefix-matched the exact detail key and silently missed the sibling endpoints (gate checks, kill-check, extractions, icp).
