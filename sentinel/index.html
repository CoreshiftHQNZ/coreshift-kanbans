esbuild banner: __dirname/__filename polyfill
phase-4infrashippedconst daemonPath = \;createRequirefileURLToPathdirnamegit diff HEAD
The hidden root cause behind Brief 9b's two earlier deploy crashes. node-cron (both v3 and v4) has `const daemonPath = \`${__dirname}/daemon.js\`;` at module top level — ReferenceError in an ESM bundle. The polyfill banner via `createRequire`/`fileURLToPath`/`dirname` was sitting **uncommitted in the local working tree** the whole time. Once committed (PR #40), Brief 9b deployed cleanly on third attempt. Lesson: when a local build passes but CI crashes the same way twice, `git diff HEAD` is the first diagnostic
