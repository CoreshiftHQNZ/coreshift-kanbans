Add a project board

Two paths. Easy way: paste the prompt below into Claude Code and let it do the work. Harder way: follow the six manual steps. Both end in the same place — a board live at coreshifthqnz.github.io/coreshift-kanbans/<project-slug>/.

🤖 Easy way — paste into Claude Code

Open Claude Code in any directory, paste this, fill in the two placeholders. Claude reads the template, drafts the config + cards, builds, and pushes.

Set up a new kanban board for <PROJECT_NAME> using the Coreshift standard format.

Repo: https://github.com/CoreshiftHQNZ/coreshift-kanbans (clone or cd into your local copy)

Steps:
1. Read tools/template/kanban.config.js and tools/template/KANBAN.md to learn the
   convention. Read sentinel/kanban.config.js + sentinel/KANBAN.md as the gold-standard
   example.
2. Copy tools/template/ to <project-slug>/ (lowercase-kebab-case slug).
3. Fill in <project-slug>/kanban.config.js: title, tagline, description, current
   phase, nextMilestone, 3 goals, links (repo, app URL, docs), 3 roles, and 3-5
   phases (typically phase-0 prove → phase-1 ship → phase-2 expand → ...).
4. Replace the example cards in <project-slug>/KANBAN.md with real ones. Read this
   project's actual source / README / recent commits to populate Done. Be honest
   about what's In Progress, Blocked, This Week, and Backlog. Card format:
   - **Title** `tag` `tag` — Description.
   Keep the five column headings exactly: ✅ Done / 🟡 In Progress / 🚫 Blocked
   / 🔵 This Week / ⚪ Backlog.
5. Run: node tools/build.js <project-slug>
   Open <project-slug>/index.html locally and check it. Refine config until
   the hero, phases, and roles read clean.
6. Edit the root index.html to add a project card linking to ./<project-slug>/,
   slotted alphabetically.
7. Commit (message: "<project-slug>: add kanban board") and push to main.
   GitHub Pages republishes in ~10s.

Context for filling in the config:
- This project is <PROJECT_CONTEXT — paste a paragraph or link to a README/spec>.
- Treat the kanban as a real working board, not marketing copy. Cards should let
  a future Claude Code session orient immediately on where the work stands.

Replace <PROJECT_NAME>, <project-slug>, and the <PROJECT_CONTEXT> block with real values before running. Claude will do the rest, including the commit + push.

🛠 Manual way — six steps

1. Copy the template

cp -r tools/template <project-slug>

2. Fill in kanban.config.js

Edit <project-slug>/kanban.config.js. Set the title, tagline, description, goals, links, roles, and phases. Inline comments in the template explain every field; sentinel/kanban.config.js is a fully-populated reference.

3. Replace the example cards

Edit <project-slug>/KANBAN.md. The card format is:

- **Title** `tag` `tag` — Description.

The five column headings must stay exactly: ✅ Done, 🟡 In Progress, 🚫 Blocked, 🔵 This Week, ⚪ Backlog — the renderer keys off the leading emoji to color the column.

4. Build it

node tools/build.js <project-slug>

That writes <project-slug>/index.html. Open it locally to check; refine the config until it looks right.

5. Link it from the root

Edit coreshift-kanbans/index.html to add a project card pointing at the new project's index.html, slotted alphabetically.

6. Commit + push

Commit and push to main. GitHub Pages republishes in ~10 seconds.

🎨 Tag colors

tools/build.js → tagClass(tag) maps tag strings to colors. Current mappings include shipped, phase-0 through phase-4, deliverable, playbook, milestone, infra, ops, recurring, and prefixes blocked-by:* and est:*. Anything else falls through to a neutral grey. Add new mappings there if you want a new tag to get its own color across every board.

🔁 Updating an existing board

Edit <project-slug>/KANBAN.md (or kanban.config.js for hero/phase changes), then:

node tools/build.js <project-slug>
# or to rebuild every project at once:
node tools/build.js

Commit, push, done.

🔒 Building locally with internal links

The published build filters out any links flagged { internal: true } in config (so the public site doesn't expose internal-only artifacts like pitch decks). To include them when building locally:

INTERNAL=1 node tools/build.js <project-slug>

The output still goes to <project-slug>/index.html — be careful not to commit that variant, or use a different output: path in the config when building locally.
