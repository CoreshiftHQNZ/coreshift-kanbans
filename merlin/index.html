M1M1-SALES-SITE

Sales / marketing site

Public site that tells the Merlin story and presents pricing. Lead capture for sales conversations, plus the entry point into self-serve purchase. Positioning + copy steered by Ricky (see “Later”). Distinct surface from the app.

M1M1-STRIPE-SEATS

Stripe seat-based billing + checkout

Per-seat recurring subscription (monthly/annual). Self-serve uses Stripe Checkout; a webhook provisions the Business + Owner on payment and sets the seat allowance. Internal/manual deals invoiced through the same account. Built rate-agnostic until the price is set. Blocked on: seat rate + Stripe access (Ricky, Do-now).

M1M1-SELFSERVE

Self-serve signup → workspace

A company signs up, pays, becomes a Business with an Owner/Admin, picks its AI platform, and lands ready to invite learners. The whole loop with no CoreShift involvement.

M1M1-INTERNAL-CONSOLE

Internal onboarding console (CoreShift-led)

A CoreShift-team surface to onboard a client manually: create the Business, capture org/industry/size, role context, tools-in-use and pain points from a kick-off meeting, set the platform, and grant seats. Seeds the org context so learners’ pathways start richer.

M1M1-INVITES

Learner invitations (both paths)

Owner/Manager invites learners by email up to the seat allowance; learner accepts via magic-link and joins the workspace. Seat reassignment when someone leaves. Shared across self-serve and internal models.

M1M1-PLATFORM

Platform selection (per org)

During onboarding, capture which AI tool the org uses — Claude / Gemini / OpenAI / Copilot. This flows everywhere downstream: Merlin’s hand-off guidance, paste tooltips, and the wording of generated prompts all tailor to it. No live integration — tailoring only.

M2M2-INTAKE

Conversational intake (save/resume)

Per-learner interview, one question at a time, feels like a chat not a form. Captures role, tools, pain points. In the internal model CoreShift seeds org context but the learner still does their own intake. Pauses and resumes without losing place.

M2M2-CAPABILITY

Capability engine — the “Trojan horse”

Infers three dimensions — operating speed, technical capability, AI understanding. Stage 1: seed from intake answers + a light disguised assessment. Stage 2: refine continuously from real task behaviour (speed, retries, tone-tweaks, scaffolding leaned on). Capability vector lives on the learner profile, re-evaluates over time, and drives scaffolding depth / prompt verbosity / step granularity. Learner can always steer pace explicitly.

M3M3-PATHGEN

Pathway generator (fully AI-generated)

Claude generates a personalised pathway per learner, fresh each time — no fixed module library — from intake + role + tools + pain points + capability vector. Autonomous guardrails: self-checks each pathway against safety/quality/tone rules, filters or regenerates risky output, ships straight to the learner; learner + Merlin can always regenerate. No human approval bottleneck.

M3M3-DELIVERY

Pathway reveal + task screen

The “this gets me” reveal, then the task screen per the prototype: why this matters for their day, the small steps, the ready-to-use prompt (Copy), the optional result peek, gentle recovery. Prompt wording tailored to the org’s platform.

M3M3-HANDOFF

⭐ The hand-off (platform-aware)

The make-or-break round-trip, already designed in the prototype: narrate → teach window-snapping → one-tap copy + unmistakable confirm + OS-aware paste help → warm “I’ll wait”. Copy/tooltips/screenshots tailor to the chosen platform. Merlin has no visibility into the tool’s session — all guidance given before they leave.

M3M3-LOOP

Return / loop-closure + events

Warm welcome-back, honest unpunished check-in (“lots / once / just tested / not yet”), celebration + next unlock per the prototype. Every step emits events (task started/completed, loop-closure outcome, confidence read, retries) feeding both the capability engine and the manager insight engine.

M4M4-INSIGHT

Org insight engine

Aggregates in-app events only per org (never conversation content) — completions, loop-closure outcomes, confidence movement, retries, Ask-Merlin frequency + topics-as-tags — and the AI writes the narrative. Powers the manager outcomes + the constructive frustration/trend detection (“several people are blocked by the CRM”).

M4M4-WEEKLY

Weekly manager note (auto, in-app)

Generated on a weekly cron and shown in the dashboard — a warm, plain-language digest, not a chart wall. In-app only for v1 (email parked). Plus the per-person + team roll-up, never ranked.

M4M4-PROGRESS

Spellbook / levels (private, non-comparable)

Levelling driven by demonstrated real use (loop-closure + confidence), non-comparable across users at the data level. No leaderboards. Magic vocabulary toggleable to plain “skills”. Reconcile with formal accreditation (Ricky, Later).

M5M5-ASK

Ask Merlin — saved + resumable

Always-available LLM conversation (Claude), saved and resumable per learner. Feeds topic-tags (never content) into the insight engine.
