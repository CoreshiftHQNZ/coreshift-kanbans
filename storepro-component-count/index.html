Done

M0SCC-1

Phase 0 prototype — DWG/DXF parser

libredwg + ezdxf pipeline. Reads DXF strict-mode; falls back to dwgread minJSON + iconv for DWG-only inputs. Validated against Sigma Healthcare DWG.

M0SCC-2

Storepro-editable component library (YAML)

Block-name → canonical-component alias table. Seeded from Sigma block names; safe-substring matcher to avoid Revit XREF false positives. Editable by Storepro admins without dev support.

M0SCC-3

Rules engine (YAML)

Counting rules (row spacer industry-standard, mesh connector 4-vs-6) + compliance flags (11m upright, cutdown protector, double-deep backstop). Pattern-based ignore for AutoCAD anonymous noise (*U/*D/*X). Captures the ruleset Ricky provided 2026-05-13.

M0SCC-4

Architectural understanding locked

Plan view → bay count by type. Elevation view → recipe per type. Storepro already produces this manually (the capacity table on every plan-view sheet). App automates + adds compliance layer.

M0SCC-5

Build proposal approved by Storepro

Phase 1 (read & count + design review): NZ$32-48k. Phase 2 (BOM → Job Coaster): NZ$8-14k. Standalone repo, independent of platform layer for now. Full doc: docs/build-proposal.md.

M0SCC-6

Repo scaffolded at CoreshiftHQNZ/storepro-component-count

main + dev + staging branches up. Phase 0 prototype ported into parser/. api/ and web/ placeholders for next iteration. README, CLAUDE.md, .gitignore in.

M0SCC-7

Kanban entry (this page)

Added to coreshift-kanbans index. Will be kept in sync as part of the work, not as a separate pass.

M1SCC-8

Sheet rasterisation pipeline (PDF → high-res PNG)

poppler's pdftoppm wrapped in parser/vision/rasterize.py. 300 DPI default, configurable per-call. Emits per-page PNGs next to the source PDF.

M1SCC-9

Bay-rectangle detector v1 (colour + OCR + scanline) — partial

Three approaches landed: hue-based (detect_bays.py), OCR-based via Tesseract (ocr_bays.py), scanline-based (segment_bays.py). First pass on Sigma plan view at ~57% recall — preprocessing tuning is next-session work. Architecture is sound; numbers improve with tuning, not refactoring.

M3SCC-16

FastAPI backend scaffolded

api/main.py with /api/health, POST /api/jobs, GET /api/jobs/{id}. DXF parse path wired end-to-end (parser → rules → flags → JSON). DWG/PDF paths return an M1-work sentinel.

M3SCC-17

Next.js frontend scaffolded

Next.js 15 + TypeScript + Tailwind 4. Single-page upload UI with component count table, rules trace, and compliance flags display. NEXT_PUBLIC_API_URL env var for the API origin.

M3SCC-19

Railway deploy config + GitHub Actions CI

nixpacks.toml installs Python 3.11 + poppler + tesseract + libredwg system deps. railway.json for the web service. CI workflow runs parser smoke test + api import check + web typecheck/build on every push to dev/staging/main.

M2SCC-14

AI design-reviewer stubbed

api/services/reviewer.py defines the interface, gates on ANTHROPIC_API_KEY, returns a placeholder until M2 wires the actual Claude vision call. Architecture sketch (request shape, model choice) captured in the module's docstring.

M1SCC-26

DWG path wired into the API (libredwg → JSON → parser)

POST /api/jobs now accepts a DWG directly. Internal pipeline: dwgread minJSON → iconv UTF-8 sanitise → JSON parser. Verified with the Sigma DWG: counts match prototype output (428 post protector, 341 beam, 324 FEP, 319 backstop, etc).

M3SCC-17b

Web UI polished for demos

Drag-and-drop upload. Sidebar with auto-refreshing recent jobs list. Detailed result view: 3-stat summary, canonical count table, per-block breakdown with unmapped flagging, rules trace, compliance flags with severity badges. Light + dark themes.

M0SCC-27

End-to-end DWG regression test passing

tests/e2e_dwg_count.sh boots the API, uploads Sigma DWG, asserts every canonical component matches ground truth — 7/7 PASS. Proves the full chain (uvicorn → multipart → libredwg → JSON parser → rules engine → response) works end-to-end.

M1SCC-28

Drawing-conventions recommendations doc

docs/drawing-conventions.md — 13 recommendations for Storepro to author drawings the app parses more reliably. Three tiers (low-effort high-impact → nice-to-have). The two biggest leverage items: consistent block library + bay-type rectangles as real blocks.

M1SCC-11

Elevation recipe extractor (per S-type)

For each S-type, parse its elevation drawing: count beam levels, OCR weight ratings, detect mesh presence (red hatch pattern), differentiate mid-frame vs end-frame, read upright spec from dimension text. Same OCR + scanline tools as the bay detector.

M1SCC-12

Overlay detector (red bracing, sprinkler beams, etc.)

Per-bay overlays that aren't in the base recipe — extra bracing in red, sprinkler beams at specific bays, cutdown post protector requirement (per the height-vs-lowest-beam rule).

M1SCC-13

End-frame sharing logic

Detect end-of-row bays geometrically; apply frame-sharing rule (row of N bays → N+1 frames, not 2N). Reads end vs mid from the elevation drawing's frame variants.

AI design-reviewer — wire the Claude vision call

Stub is in place at api/services/reviewer.py. Wire the actual Claude vision request: sheet PNGs + parsed count + rules trace as input, structured commentary as output. Gated on ANTHROPIC_API_KEY at deploy time.
